# モジュール要件（和訳）

## モジュールについて

25%のプロジェクトを達成したことをお祝いします！

基本的な機能的なウェブサイトが整ったので、次のステップはさらなる改善のためのモジュールを選択することです。

**100%のプロジェクト完成度を達成するには、最低7つのMajorモジュールが必要です。**

各モジュールがベースラインのウェブサイトに変更を加える可能性があるため、この要件全体を注意深く確認することが重要です。したがって、この要件全体を徹底的に読むことを強く推奨します。

## ライブラリ/ツール使用に関する制約

### ❌ 禁止事項

• **モジュール全体または機能全体を即座に完全に解決するライブラリやツールの使用は禁止されています。**

### ✅ 必須遵守事項

• **サブジェクトで明示的に指示されているライブラリやツールの使用（can, must, can't）に関する直接的な指示は、必ず従う必要があります。**

### ✅ 許可事項

• **小さなライブラリやツールで、単一の簡単なタスクを解決し、より大きな機能やモジュールのサブコンポーネントを表すものの使用は許可されています。**

### 📋 評価時の説明責任

• **評価中、チームはサブジェクトで明示的に承認されていない、かつサブジェクトの制約に矛盾しないライブラリやツールの使用について正当化する必要があります。**

• **評価中、評価者は特定のライブラリやツールの使用が正当（許可される）か、それとも本質的に機能全体やモジュール全体を解決するもの（したがって禁止される）かを判断します。**

## モジュール選択の自由

要件リストに記載されているモジュールは**選択肢**であり、全てを実装する必要はありません。

### 例:
- Blockchain Score Storage → 実装しなくてもOK（選択しない）
- Remote Authentication (OAuth) → 実装しなくてもOK（選択しない）
- Multiplayer (>2 players) → 実装しなくてもOK（選択しない）
- Another Game → 実装しなくてもOK（選択しない）
- WAF/ModSecurity + Vault → 実装しなくてもOK（選択しない）
- Log Management Infrastructure → 実装しなくてもOK（選択しない）
- Microservices Architecture → 実装しなくてもOK（選択しない）

**重要なのは、最低7つのMajorモジュールを実装することです。**

現在の実装は**7つのMajorモジュール**を満たしているため、要件を達成しています。

## ライブラリ使用の判断基準

### ✅ 許可される使用例

1. **bcryptjs** - パスワードハッシュ化のみ（認証モジュール全体を解決しない）
2. **speakeasy** - TOTP生成のみ（2FAモジュール全体を解決しない）
3. **qrcode** - QRコード生成のみ（2FAモジュール全体を解決しない）
4. **Socket.IO** - WebSocket通信のみ（Remote Playersモジュール全体を解決しない）
5. **Tailwind CSS** - スタイリングのみ（フロントエンド全体を解決しない）

### ❌ 禁止される使用例

1. **完全な認証ライブラリ** - 認証モジュール全体を解決する
2. **完全なチャットライブラリ** - チャットモジュール全体を解決する
3. **完全なゲームエンジン** - ゲームモジュール全体を解決する
